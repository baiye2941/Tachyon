# Spec Brief：H-02 暂停/恢复线性化 · M-03 进度 delta 不丢

## 范围
1. **H-02**：`DownloadSupervisor` 每 task 串行锁；`pause_task_inner` / `resume_task_inner` / `cancel` 控制路径在锁内完成「改 TaskInfo + send_command」。
2. **H-02**：`resume` 若 `has_running_task` 但 `send_command` 失败（receiver 已关），必须 `restart_download`，不得伪成功。
3. **M-03**：`ProgressBroker` 将 watch 改为 **broadcast（容量≥64）** 承载完整 `ProgressEvent`；连续 send 不互相覆盖。`subscribe_progress` 用 `recv` 顺序消费。
4. 测试：H-02 串行锁下 pause/resume 最终一致；M-03 连续两次带不同 completed_delta 的事件均可被订阅者收到。
