# Spec Brief：SEC-002 取消 create_task 自动授权 · SEC-006 备份路径白名单

## 范围
1. **SEC-002**：`TaskService::create_task` 移除 `download_dir.is_some() → validate_explicit + authorized_dirs.push` 自动授权；未授权目录一律拒绝
2. 新增 IPC `authorize_download_directory(path, confirmation_token)`：需 `request_confirmation(action="authorize_download_directory")` 后才把目录加入 `authorized_dirs` 并持久化
3. **前端**：`NewTaskModal` 在 dialog 选目录后调用 authorize（确认 token）
4. **SEC-006**：`export_backup`/`import_backup` 路径必须位于 `authorized_dirs` 之下（或与已授权根同前缀），禁止任意路径写

## 非目标
- 彻底解决 SEC-003（XSS 与前端同权仍可 request_confirmation）——仅提高破坏性操作门槛
- 改 Tauri capability 模型
