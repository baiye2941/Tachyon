# Spec Brief：SEC-010 中间路径组件 · SEC-007 代理 SSRF 边界 · SEC-013 release smoke 硬门禁

## 范围
1. **SEC-010（可落地加固）**：`validate_save_path` / 授权目录链在 base→目标 路径上对每个**已存在**中间组件做 `symlink_metadata` 拒绝 symlink/reparse；open 前再验一次。不宣称已实现 openat2 句柄化打开。
2. **SEC-007（诚实边界）**：文档 + 设置页提示：启用 HTTP/SOCKS 代理后，目标域名解析与可达性由代理决定，本地 `PublicDnsResolver`/`reject_forbidden_ip` **不覆盖**代理后端。
3. **SEC-013（最小）**：`install-smoke` 去掉 job 级 `continue-on-error`；**本平台产物存在 + 包结构**硬失败；GUI 相关步骤若保留则独立 soft 或改为 warning 但不掩盖产物缺失。

## 非目标
- 完整 openat2 / Windows 相对句柄打开
- 客户端代理解析目标 IP 的主动 SSRF 拦截（需真实 proxy 测试矩阵）
- Authenticode / Tauri updater 签名供应链
